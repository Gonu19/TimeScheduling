### [API 2.1] Update availabilities (`PUT /api/sessions/{sessionId}/members/{memberId}/availabilities)

**▼ Header**

- `Content-Type: application/json` : mandatory (본 API는 Network Payload 최적화를 위해 Bitmask Integer 배열 형태의 JSON 규격을 엄격히 요구합니다.)
    

**▼ Request**

- **Path Variable**
    
    - `sessionId` (string, UUID 형식): 접속 중인 일정 조율 방의 고유 ID
        
    - `memberId` (long): 가용시간을 등록/수정하는 주체인 팀원의 Primary Key(고유 식별자)
        
- **Request Body (object)**: 팀원의 주간 가용시간 데이터
    
    - `availableBitmasks` (array of long): 하루별 가용시간을 압축한 64-bit 정수 배열. (시작날짜가 index 0, 끝 날짜가 index 6) 하루 24시간을 30분 단위(총 48 슬롯)로 분할하여 Binary(이진수)로 매핑한 뒤 Long 타입으로 변환한 값입니다.
        

JSON

```
{
  "availableBitmasks": [
    281474976710655, 
    0,               
    4294967295,      
    0,
    1048575,         
    0,
    0                
  ]
}
```

**▼ Response** * **Availability 메타 데이터 (object)**: 성공적으로 Database에 Persist(영속화)된 가용시간 정보

* `availabilityId` (long): 저장된 가용시간 데이터의 고유 식별자

* `updatedAt` (string): 최종 등록 및 수정 일시 (ISO 8601 Format)

JSON

```
{
    "availabilityId": 42,
    "updatedAt": "2026-05-26T18:30:45"
}
```

**▼ Error**

- `errorCode` (string): 시스템 에러 코드
    
- `message` (string): 에러 상세 메시지
    

JSON

```
{
    "errorCode": "ERR_INVALID_BITMASK_PAYLOAD",
    "message": "비트마스크 배열의 크기는 반드시 7이어야 하며, 48-bit 허용 범위를 초과할 수 없습니다."
}
```

**▼ 예외 (Exception Handling)**

- **Validation 실패 (Bad Request)**: 프론트엔드에서 넘어온 `availableBitmasks` 배열의 Length(길이)가 7이 아니거나, 각 요소가 $2^{48}-1$ 을 초과하는 Invalid value(유효하지 않은 값)일 경우 **400 Bad Request** 반환.
    
- **권한 및 정합성 충돌 (Forbidden)**: URL의 `sessionId` 방에 속하지 않은 `memberId`가 악의적으로 Data update를 시도하는 Cross-reference(교차 참조) 감지 시 **403 Forbidden** 반환. (데이터 무결성 방어)
    
- **리소스 부재 (Not Found)**: 입력된 `sessionId`나 `memberId`가 DB에 존재하지 않거나 Purge(삭제)된 상태일 경우 **404 Not Found** 반환.
    
- **서버 에러 (Internal Server Error)**: Transaction Commit 중 DB Lock이나 내부 장애 발생 시 **500 Internal Server Error** 반환.
