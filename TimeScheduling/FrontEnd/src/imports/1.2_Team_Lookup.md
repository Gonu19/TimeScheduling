### [API 1.2] Team Lookup (GET `/api/sessions/{sessionId}`)

▼ **Header**

- `Content-Type: application/json` : mandatory _(본 서비스는 UUID를 통한 익명 접속을 지원하므로 JWT 토큰 등의 인증 헤더는 요구하지 않습니다.)_
    

▼ **Request**

- Path Variable
    
    - `sessionId` (string, UUID 형식): 접속하려는 일정 조율 방의 고유 ID
        

▼ **Response** 세션(팀) 정보 (object): 해당 방의 렌더링에 필요한 기본 메타 데이터

- `title` (string): 일정 조율 방의 이름
    
- `domainType` (string): 팀의 목적 (`"MEETING"` 또는 `"WORK"`)
    
- `status` (string): 해당 방의 현재 상태 (`"OPEN"`: 진행 중, `"CONFIRMED"`: 확정 완료)
    

JSON

```
{
    "title": "소프트웨어공학 팀플 회의",
    "domainType": "MEETING",
    "status": "OPEN"
}
```

▼ **Error**

- `errorCode` (string): 시스템 에러 코드
    
- `message` (string): 에러 상세 메시지
    

JSON

```
{
    "errorCode": "ERR_SESSION_NOT_FOUND",
    "message": "존재하지 않거나 만료된 세션입니다."
}
```

▼ **예외**

1. **세션 없음 (Not Found):** URL에 포함된 `sessionId`가 DB에 없거나, 7일이 경과하여 자동 삭제(Purge)된 방일 경우 `404 Not Found` 에러 반환. (프론트엔드는 이 경우 "만료된 방입니다" 안내 페이지로 리다이렉트 처리)
    
2. **서버 에러:** 서버 내부 처리 중 오류 발생 시 `500 Internal Server Error` 반환.
