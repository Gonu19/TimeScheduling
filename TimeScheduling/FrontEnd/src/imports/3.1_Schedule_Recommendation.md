### [API 3.1] Schedule Recommendation (GET `/api/sessions/{sessionId}/recommendations`)

▼ **Header**

- `Content-Type: application/json` : mandatory
    
- `Authorization: Bearer <adminToken>` : **mandatory** (방장 권한 검증 및 IDOR 취약점 방어용)
    

▼ **Request**

- Path Variable
    
    - `sessionId` (string, UUID 형식): 추천안을 도출할 일정 조율 방의 고유 ID
        

▼ **Response** 백엔드 인메모리 연산을 통해 도출된 3가지 스케줄 추천안 (object)

- `version` (integer): Optimistic Locking(낙관적 락) 검증을 위한 현재 데이터베이스의 Session 엔티티 버전
    
- `recommendations` (array of objects): 도출된 추천안 리스트
    
    - `rank` (integer): 추천 순위 (1~3)
        
    - `recommendationType` (string): 추천 전략 (`"MAX_ATTENDANCE"`, `"EQUAL_DISTRIBUTION"`, `"MAX_CONTINUITY"`)
        
    - `description` (string): 해당 추천안의 특징을 설명하는 요약 텍스트 
        
    - `date`(string) : 시작된 정확한 날짜
        
    - `startTime` (string): 제안 시작 시간 (Format: `HH:mm`)
        
    - `endTime` (string): 제안 종료 시간 (Format: `HH:mm`)
        
    - `attendanceRate` (integer): 참석 가능 인원 비율 (%)
        
    - `assignedMembers` (array of strings): 해당 시간에 참석/배정 가능한 멤버 이름 목록
        
    - `missingMembers` (array of strings): 해당 시간에 참석 불가능한 멤버 이름 목록
        


JSON

```
{
  "version": 3,
  "recommendations": [
    {
      "rank": 1,
      "recommendationType": "MAX_ATTENDANCE",
      "description": "최대 인원이 참석할 수 있는 시간대입니다.",
      "date": "2026-06-12", // ⬅️ [NEW] 추천된 정확한 날짜
      "startTime": "14:00",
      "endTime": "16:00",
      "attendanceRate": 100,
      "assignedMembers": ["강은우", "김백엔드"],
      "missingMembers": []
    }
  ]
}
```

▼ **Error**

- `errorCode` (string): 시스템 에러 코드
    
- `message` (string): 에러 상세 메시지
    


JSON

```
{
    "errorCode": "ERR_MANDATORY_MEMBER_MISSING",
    "message": "필수 참석 인원(isMandatory: true) 중 아직 가용시간을 입력하지 않은 멤버가 존재하여 알고리즘 연산을 수행할 수 없습니다."
}
```

▼ **예외 (Exception Handling)**

1. **권한 없음 (Unauthorized):** 헤더에 `adminToken`이 누락되었거나 해당 `sessionId`의 마스터 토큰과 일치하지 않을 경우 **401 Unauthorized** 반환.
    
2. **미입력 인원 (Conflict):** 알고리즘 연산의 Constraint(제약 조건)인 `isMandatory` 멤버 중 단 한 명이라도 가용시간을 제출하지 않은 상태라면 연산을 Hard Block 처리하고 **409 Conflict** 반환.
    
3. **서버 에러:** Bitwise operation(비트 연산) 도중 OutOfBounds 예외 등 내부 로직 결함 발생 시 **500 Internal Server Error** 반환.
