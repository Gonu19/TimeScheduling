
### [API 2.2] Load Member Availability (GET `/api/sessions/{sessionId}/members/{memberId}/availabilities`)

▼ **Header**

- `Content-Type: application/json` : mandatory
    

▼ **Request**

- Path Variable
    
    - `sessionId` (string, UUID 형식): 현재 접속 중인 일정 조율 방의 고유 ID
        
    - `memberId` (long): **가용시간을 불러오고자 하는 대상(Target) 멤버의 Primary Key** _(예: 내가 '강은우'지만, '김백엔드'의 시간표를 복사하고 싶다면 김백엔드의 ID를 삽입)_
        

▼ **Response** 대상 멤버의 가용시간 비트마스크 데이터 (object)

- `availableBitmasks` (array of long): 대상 멤버가 기존에 등록해 둔 7개의 64-bit 정수 배열. (프론트엔드는 이 배열을 받아 그대로 Grid UI State에 덮어씌웁니다.)
    

**JSON**

JSON

```
{
    "availableBitmasks": [
        281474976710655, 
        0,               
        4294967295,      
        0,
        1048575,         
        0,
        0                
    ]
}
```

▼ **Error**

- `errorCode` (string): 시스템 에러 코드
    
- `message` (string): 에러 상세 메시지
    

**JSON**

JSON

```
{
    "errorCode": "ERR_AVAILABILITY_NOT_FOUND",
    "message": "해당 멤버는 아직 가용시간을 등록하지 않았습니다."
}
```

▼ **예외 (Exception Handling)**

1. **데이터 없음 (Not Found):** URL에 명시된 `memberId`가 아직 단 한 번도 가용시간(2.1 API)을 제출하지 않은 상태일 경우 `404 Not Found` 에러 반환. _(프론트엔드는 이 에러를 캐치하여 "아직 시간표를 등록하지 않은 멤버입니다"라는 Toast/Alert를 띄웁니다.)_
    
2. **잘못된 접근 (Not Found / Forbidden):** `sessionId`가 존재하지 않거나, 해당 `memberId`가 그 세션에 속한 멤버가 아닐 경우 `404` 또는 `403` 에러 반환.
    
3. **서버 에러:** 데이터베이스 Read 과정에서 예외 발생 시 `500 Internal Server Error` 반환.
    

### 💡 프론트엔드 연동 꿀팁 (Frontend Integration Guide)

이 API의 핵심은 Data Fetching(데이터 조회)을 하자마자 추가적인 가공 없이 바로 프론트엔드의 `availableBitmasks` State로 치환할 수 있다는 점입니다.

1. 사용자가 드롭다운에서 '김백엔드(memberId: 102)'를 선택합니다.
    
2. 프론트엔드는 `GET /api/sessions/550e.../members/102/availabilities`를 호출합니다.
    
3. 응답받은 배열을 React의 `setAvailableBitmasks(response.data.availableBitmasks)`로 업데이트합니다.
    
4. 화면의 30분 단위 Grid가 김백엔드의 시간표로 쫙 칠해집니다.
    
5. 유저가 약간 수정(Customizing)을 거친 뒤 최종 등록을 누르면, [API 2.1]을 호출하여 자신의 `memberId`로 `PUT` 요청을 보냅니다.
