### [API 4.1] 최종 확정 스케줄 조회 (GET `/api/sessions/{sessionId}/result`)

▼ **Header**

- `Content-Type: application/json` : mandatory
    

▼ **Request**

- **Path Variable**
    
    - `sessionId` (string, UUID 형식): 최종 확정 결과를 조회하고자 하는 일정 조율 방의 고유 ID
        

▼ **Response** 데이터베이스에 _Persist_(영속화)된 최종 확정 스케줄 데이터 (object)

- `confirmedType` (string): 최종 확정된 추천안 알고리즘 전략 타입 (`"MAX_ATTENDANCE"`, `"EQUAL_DISTRIBUTION"`, `"MAX_CONTINUITY"`)
    
- `date`(string) : 시작된 정확한 날짜
    
- `startTime` (string): 확정된 이벤트 시작 시간 (Format: `HH:mm`)
    
- `endTime` (string): 확정된 이벤트 종료 시간 (Format: `HH:mm`)
    
- `attendees` (array of strings): 해당 타임 슬롯에 최종 배정된 고정 참석자 명단
    

**JSON**

JSON

```
{
  "confirmedType": "MAX_ATTENDANCE",
  "date": "2026-06-12", 
  "startTime": "14:00",
  "endTime": "16:00",
  "attendees": ["강은우", "김백엔드"]
}
```

▼ **Error**

- `errorCode` (string): 시스템 에러 코드
    
- `message` (string): 에러 상세 메시지
    

**JSON**

JSON

```
{
  "errorCode": "ERR_RESULT_NOT_FOUND",
  "message": "아직 방장에 의해 스케줄이 확정되지 않은 세션입니다."
}
```

▼ **예외 (Exception Handling)**

1. **결과 부재 (Not Found):** 방의 상태(`status`)가 아직 `"OPEN"`이거나 방장이 확정 API(3.2)를 호출하지 않아 최종 스케줄 엔티티가 생성되지 않은 경우 **404 Not Found** 반환.
    
2. **세션 만료 (Not Found):** 존재하지 않거나 유효기간 경과로 인해 _Purge_(삭제)된 `sessionId`로 접근할 경우 **404 Not Found** 반환.
