[[데이터베이스]]
본 시스템은 회원가입 없는 익명 기반의 팀 협업 스케줄링 서비스로, 데이터 최적화와 무결성을 위해 비트마스킹(64-bit Integer) 및 낙관적 락(@Version) 제약 조건을 반영하여 설계되었습니다.

## 1. session (익명 일정 세션 테이블)

- **역할**: 사용자들이 일정을 맞추기 위해 생성하는 가상의 '회의실/근무 배정 방' 정보 자체를 관리합니다.
    

| **컬럼명**        | **데이터 타입**     | **제약 조건**     | **역할 및 설명**                                                                                                                                                              |
| -------------- | -------------- | ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **session_id** | `VARCHAR(36)`  | PRIMARY KEY   | 방의 고유 식별자입니다. 보안과 익명성을 위해 숫자가 아닌 추측 불가능한 36자리 UUID 형태로 저장하여 타인이 주소를 유추해 무단 진입하는 것을 방어합니다.                                                                                |
| **title**      | `VARCHAR(100)` | NOT NULL      | 방의 제목입니다. (예: "캡스톤 기획 회의", "A파트 주말 야간 근무 배정")                                                                                                                            |
| **purpose**    | `VARCHAR(20)`  | NOT NULL      | 세션의 목적입니다. 알고리즘 엔진이 연산 기준을 판단할 수 있도록 '회의(MEETING)' 또는 '근무(WORK)' 값을 가집니다.                                                                                                |
| **start_date** | `DATE`         | NOT NULL      | **[변경됨]** 조율의 기준이 되는 **'시작 날짜(Start Date)'**입니다. 백엔드는 이 날짜를 기준으로 자동으로 향후 7일(Start Date + 6일)의 기간을 계산하여 가용시간 비트마스크 배열과 1:1 매핑(Mapping)합니다. 방장의 인지 부하를 줄이고 데이터 구조를 최적화합니다. |
| **created_at** | `DATETIME`     | DEFAULT NOW() | 방이 개설된 시간입니다. DB 내부의 Event Scheduler가 이 시간을 기준으로 7일이 지난 만료 세션을 추적하여 자율 삭제(Purge)합니다.                                                                                     |
| **status** | `VARCHAR(20)`     | NOT NULL, DEFAULT 'OPEN' | 세션의 현재 상태입니다. 'OPEN'(조율 중), 'CONFIRMED'(확정 완료) 등의 값을 가지며 프론트엔드의 화면 렌더링 및 입력 가능 여부를 제어합니다. |
| **confirmed_blocks** | `TEXT`     | Nullable | 방장이 스케줄을 최종 확정(CONFIRMED)했을 때, 해당 스케줄 결과 데이터 배열(TimeBlock 리스트)이 JSON 형태로 직렬화되어 저장됩니다. |
| **admin_token** | `VARCHAR(36)`     | Nullable | 세션 생성 시 방장에게만 발급되는 고유 관리자 권한 인증용 토큰입니다. |
| **requirements_json** | `TEXT`     | Nullable | **[근무 요건 데이터]** WORK 도메인인 경우, 각 요일별 필요 인력 및 근무 시간 조건 등이 JSON 포맷으로 저장됩니다. 추천 알고리즘 엔진이 이 값을 읽어들여 최적의 근무표를 도출합니다. |

## 2. participant (참가자 정보 테이블)

- **역할**: 방에 참여하는 개별 사용자들의 식별 정보를 관리합니다. 동명이인 문제를 해결하고 익명 수정 권한을 검증하기 위해 분리되었습니다.
    

|**컬럼명**|**데이터 타입**|**제약 조건**|**역할 및 설명**|
|---|---|---|---|
|**participant_uid**|`VARCHAR(36)`|PRIMARY KEY|사용자의 고유 식별자(UUID)입니다. 가입이 없는 익명 시스템이므로, 접속 시 브라우저에 임시 발급되는 고유 키를 저장하여 동명이인을 완벽히 구분합니다.|
|**nickname**|`VARCHAR(50)`|NOT NULL|화면에 표시될 사용자의 이름 또는 익명 닉네임입니다. 중복 작성이 가능합니다.|
|**created_at**|`DATETIME`|DEFAULT NOW()|사용자가 방에 처음 참여하여 닉네임을 등록한 시간입니다. 화면 렌더링 시 정렬 안정성(Sorting stability)을 보장하고, 알고리즘 동점자 발생 시 선착순 배정을 위한 Heuristic(발견적) 기준으로 활용됩니다.|

## 3. participant_schedule (가용 시간 및 동시성 관리 테이블)

- **역할**: 사용자가 그리드 드래그를 통해 제출한 시간표 데이터를 저장하고, 기습적인 수정으로 인한 데이터 오염을 방지하는 핵심 비즈니스 테이블입니다.
    

| **컬럼명**                       | **데이터 타입**    | **제약 조건**                                        | **역할 및 설명**                                                                                                                                                                 |
| ----------------------------- | ------------- | ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **schedule_id**               | `BIGINT`      | PRIMARY KEY, AUTO_INCREMENT                      | 제출된 시간표 데이터 자체의 고유 순번(ID)입니다.                                                                                                                                               |
| **session_id**                | `VARCHAR(36)` | FOREIGN KEY<br><br>  <br><br>(ON DELETE CASCADE) | 이 시간표가 어느 '방(session)'에 속해 있는지를 연결하는 고리입니다. 세션이 폭파되면 해당 시간표도 함께 자동 삭제됩니다.                                                                                                   |
| **participant_uid**           | `VARCHAR(36)` | FOREIGN KEY<br><br>  <br><br>(ON DELETE CASCADE) | 이 시간표를 제출한 '참가자(participant)'가 누구인지 연결하는 고리입니다. 참가자 정보가 지워지면 시간표도 함께 자동 삭제됩니다.                                                                                              |
| **bitmask_0** ~ **bitmask_6** | `BIGINT`      | NOT NULL, DEFAULT 0                              | **[비트 연산 최적화]** 일주일(7일) 치 가용 시간을 요일별로 나누어 저장하는 7개의 Column입니다. 하루 24시간을 30분 단위로 쪼갠 총 48개의 타임 슬롯 정보를 64-bit 정수로 압축 저장하며, O(1)에 가까운 초고속 Bitwise operation(비트 논리 연산)을 가능하게 합니다. |
| **version**                   | `INT`         | NOT NULL, DEFAULT 0                              | **[낙관적 락 제어]** 관리자가 스케줄을 최종 확정하는 순간, 다른 참가자가 가용 시간을 기습적으로 수정하여 추천 알고리즘의 결과가 오염되는 것을 막기 위한 Concurrency control(동시성 제어) 필드입니다. JPA의 @Version과 연동되어 충돌을 튕겨냅니다.                 |
