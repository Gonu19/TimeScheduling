### [API 1.4] Shift Requirement Registration (PUT /api/sessions/{sessionId}/requirements)

**▼ Header**

- `Content-Type: application/json` : mandatory
    
- `Authorization: Bearer {Admin_Token}` : mandatory (1.3 API에서 발급받은 관리자 세션 토큰)
    

**▼ Request**

- **Path Variable**
    
    - `sessionId` (string, UUID 형식): 근무 요건을 등록/수정하고자 하는 일정 조율 방의 고유 ID
        
- **Request Body (Payload)**
    
    - `requirementData` (object): 7일간의 요일별(`day_0` ~ `day_6`) 필요 인원(Required Headcount) 데이터를 담은 JSON 객체. 각 요일은 하루 24시간을 30분 단위로 나눈 48개의 정수(Integer) 배열로 구성됨.
      > **[CRITICAL WARNING] Timezone 정책:** 해당 48개 슬롯 배열은 방 생성 기준이 되는 Local Time(예: KST)만을 철저히 기준으로 하며, UTC 변환을 절대 거치지 않아야 합니다. (Timezone 오프셋으로 인한 밀림 현상 방지)
        
- **JSON**
    
```JSON
{
  "requirementData": {
    "day_0": [0, 0, 0, 2, 2, 2, 0, 0], // (실제로는 48개 길이의 정수 배열)
    "day_1": [0, 0, 0, 1, 1, 1, 0, 0],
    "day_2": [0, 0, 0, 0, 0, 0, 0, 0],
    "day_3": [0, 0, 0, 2, 2, 2, 0, 0],
    "day_4": [0, 0, 0, 1, 1, 1, 0, 0],
    "day_5": [0, 0, 0, 0, 0, 0, 0, 0],
    "day_6": [0, 0, 0, 0, 0, 0, 0, 0]
  }
}
```

**▼ Response**

- **등록 결과 데이터 (object)**
    
    - `success` (boolean): 근무 요건이 DB에 성공적으로 반영(Update)되었는지 여부
        
    - `message` (string): 처리 결과에 대한 안내 메시지
        
- **JSON**
    

JSON

```
{
  "success": true,
  "message": "근무 요건이 성공적으로 등록되었습니다."
}
```

**▼ Error**

- `errorCode` (string): 시스템 에러 코드
    
- `message` (string): 에러 상세 메시지
    
- **JSON (예시: 인증 실패 시)**
    

JSON

```
{
  "errorCode": "ERR_UNAUTHORIZED_ADMIN",
  "message": "관리자 권한이 없거나 토큰이 유효하지 않습니다."
}
```

**▼ 예외 (Exception Handling)**

- **권한 없음 (Unauthorized):** `Authorization` 헤더가 누락되었거나, 조작된(Invalid) 토큰으로 접근할 경우 시스템 보안을 위해 **401 Unauthorized** 반환.
    
- **형식 오류 (Bad Request):** `requirementData` 내의 배열 길이가 48이 아니거나, 배열 내에 0 미만의 음수가 들어오는 등 데이터 무결성(Data Integrity)에 어긋나는 Payload가 인입될 경우 **400 Bad Request** 반환.
    
- **세션 부재 (Not Found):** 존재하지 않거나 유효기간 경과로 인해 삭제된 sessionId로 접근할 경우 **404 Not Found** 반환.
    
- **상태 충돌 (Conflict):** 이미 관리자에 의해 최종 스케줄이 '확정(CONFIRMED)'된 방에서 뒤늦게 근무 요건을 수정하려 시도할 경우, 비즈니스 로직 방어를 위해 **409 Conflict** 반환.
