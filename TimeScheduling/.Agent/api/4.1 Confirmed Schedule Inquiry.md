### [API 4.1] 최종 확정 스케줄 조회 (GET `/api/sessions/{sessionId}/result`)

▼ **Header**

- `Content-Type: application/json` : mandatory
    

▼ **Request**

- **Path Variable**
    
    - `sessionId` (string, UUID 형식): 최종 확정 결과를 조회하고자 하는 일정 조율 방의 고유 ID
        

▼ **Response** 데이터베이스에 _Persist_(영속화)된 최종 확정 스케줄 데이터 (object)

- `confirmedType` (string): 최종 확정된 추천안 알고리즘 전략 타입 (`"MAX_ATTENDANCE"`, `"EQUAL_DISTRIBUTION"`, `"MAX_CONTINUITY"`)
    
- `startTime` (string): 확정된 이벤트 시작 시간 (Format: `HH:mm`)
    
- `endTime` (string): 확정된 이벤트 종료 시간 (Format: `HH:mm`)
    
- `attendees` (array of strings): 해당 타임 슬롯에 최종 배정된 고정 참석자 명단
    

**JSON**

JSON

```
{
  "confirmedType": "MAX_ATTENDANCE",
  "startTime": "14:00",
  "endTime": "16:00",
  "attendees": [
    "강은우",
    "김기획",
    "박프론트"
  ]
}
```

▼ **Error**

- `errorCode` (string): 시스템 에러 코드
    
- `message` (string): 에러 상세 메시지
    

**JSON**

JSON

```
{
  "errorCode": "ERR_RESULT_NOT_FOUND",
  "message": "아직 방장에 의해 스케줄이 확정되지 않은 세션입니다."
}
```

▼ **예외 (Exception Handling)**

1. **결과 부재 (Not Found):** 방의 상태(`status`)가 아직 `"OPEN"`이거나 방장이 확정 API(3.2)를 호출하지 않아 최종 스케줄 엔티티가 생성되지 않은 경우 **404 Not Found** 반환.
    
2. **세션 만료 (Not Found):** 존재하지 않거나 유효기간 경과로 인해 _Purge_(삭제)된 `sessionId`로 접근할 경우 **404 Not Found** 반환.
    

### [API 4.2] 스케줄 긴급 수정 / Manual Override (PATCH `/api/sessions/{sessionId}/result`)

▼ **Header**

- `Content-Type: application/json` : mandatory
    
- `Authorization: Bearer <adminToken>` : **mandatory** (방장 권한 검증 및 자원 보호용)
    

▼ **Request**

- **Path Variable**
    
    - `sessionId` (string, UUID 형식): 수동 보정을 적용할 일정 조율 방의 고유 ID
        
- **Request Body (object)**: 관리자가 개입하여 강제로 변경한 타임 슬롯 및 사유 데이터
    
    - `adjustedSlots` (array of objects): 수동 변경 내역 리스트
        
        - `time` (string): 변경 대상 타임 슬롯 (Format: `HH:mm`)
            
        - `removed` (string, optional): 해당 슬롯에서 제외할 팀원 이름
            
        - `added` (string, optional): 해당 슬롯에 대체 투입할 팀원 이름
            
    - `overrideReason` (string): **[필수] 데이터 무결성 방어 및 감사 로그(Audit Log) 축적을 위한 강제 수정 사유**
        

**JSON**

JSON

```
{
  "adjustedSlots": [
    {
      "time": "14:00",
      "removed": "강은우",
      "added": "박프론트"
    }
  ],
  "overrideReason": "강은우 병가로 인한 당일 긴급 교체 투입"
}
```

▼ **Response** 성공적인 수동 보정 및 업데이트 결과 (object)

- `status` (string): 리소스 엔티티의 최신 상태
    
- `message` (string): 작업 완료 안내 메시지
    

**JSON**

JSON

```
{
  "status": "UPDATED",
  "message": "스케줄이 관리자에 의해 수동 수정되었습니다."
}
```

▼ **Error**

- `errorCode` (string), `message` (string)
    

**JSON**

JSON

```
{
  "errorCode": "ERR_REASON_REQUIRED",
  "message": "스케줄 강제 변경 시, 감사 로그를 위한 수정 사유(overrideReason)는 비어둘 수 없습니다."
}
```

▼ **예외 (Exception Handling)**

1. **권한 오류 (Unauthorized):** 헤더의 `adminToken`이 누락되었거나 마스터 토큰과 불일치할 경우 무단 접근으로 판단하여 **401 Unauthorized** 반환.
    
2. **유효성 검증 실패 (Bad Request):** `overrideReason` 필드가 공백(Blank)이거나 `adjustedSlots` 배열 형식이 규격 외의 _Invalid Value_일 경우 **400 Bad Request** 반환.
    
3. **선행 조건 미충족 (Conflict):** 아직 확정(Confirmed)되지도 않은 스케줄에 대해 수정 조작을 시도하는 불법적인 멱등성(_Idempotency_) 위반 발생 시 **409 Conflict** 반환.
