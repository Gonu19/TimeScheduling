### [API 3.2] 스케줄 최종 확정 (POST /api/sessions/{sessionId}/confirm)

**▼ Header**

- `Content-Type: application/json` : mandatory
    
- `Authorization: Bearer <adminToken>` : mandatory
    

**▼ Request**

- **Path Variable**
    
    - `sessionId` (string, UUID 형식): 확정할 세션 고유 ID
        
- **Request Body (object)**
    
    - `confirmedBlocks` (array of objects): 방장이 최종 선택하거나 수동 조정한 다중 근무/회의 구간(TimeBlock) 리스트.
        
        - `startTime` (string, ISO 8601): 구간 시작 일시 (예: "2026-05-29T17:30:00")
            
        - `endTime` (string, ISO 8601): 구간 종료 일시 (예: "2026-05-29T18:00:00")
            
        - `assignedWorkers` (array of objects, Optional): **WORK 도메인의 경우** 해당 타임 블록에 배정된 근무자 명단 리스트. (회의의 경우 생략될 수 있음)
            - `id` (string): 참가자 고유 식별자
            - `name` (string): 참가자 이름
            - `isMandatory` (boolean): 필참 인력 여부
            
    - `version` (integer): `[API 3.1]` 또는 `[API 3.3]`에서 응답받았던 데이터 버전 (Concurrency control 용도)
        
- **JSON**
    

JSON

```json
{
    "confirmedBlocks": [
        {
            "startTime": "2026-05-29T17:30:00",
            "endTime": "2026-05-29T18:00:00",
            "assignedWorkers": [
                {
                    "id": "a1b2c3d4",
                    "name": "고은우",
                    "isMandatory": true
                },
                {
                    "id": "e5f6g7h8",
                    "name": "김알바",
                    "isMandatory": false
                }
            ]
        },
        {
            "startTime": "2026-05-30T17:00:00",
            "endTime": "2026-05-30T17:30:00",
            "assignedWorkers": [
                {
                    "id": "i9j0k1l2",
                    "name": "박프론트",
                    "isMandatory": false
                }
            ]
        }
    ],
    "version": 3
}
```

**▼ Response**

- **성공적인 확정 처리 결과 (object)**
    
    - `status` (string): 세션의 변경된 상태
        
    - `message` (string): 클라이언트에 노출할 성공 메시지
        
- **JSON (200 OK)**
    

JSON

```
{
    "status": "CONFIRMED",
    "message": "스케줄이 최종 확정되었습니다."
}
```

**▼ Error**

- `errorCode` (string): 시스템 에러 코드
    
- `message` (string): 에러 상세 메시지
    
- **JSON**
    

JSON

```
{
    "errorCode": "ERR_CONCURRENCY_CONFLICT",
    "message": "추천안을 검토하는 동안 다른 팀원이 가용시간을 수정했습니다. 최신 데이터를 기반으로 추천안을 다시 도출해주세요."
}
```

**▼ 예외 (Exception Handling)**

- **동시성 충돌 (Conflict):** 클라이언트가 보낸 `version`과 DB에 Persist(영속화)된 Session 엔티티의 `@Version` 값이 불일치할 경우, JPA `OptimisticLockException`을 캐치하여 **409 Conflict** 반환. (Data integrity 무결성 보장)
    
- **상태 이상 (Bad Request):** 이미 "CONFIRMED" 상태인 방에 대해 중복 확정 요청을 보낼 경우 **400 Bad Request** 반환.
    
- **데이터 형식 오류 (Bad Request):** `confirmedBlocks` 배열이 비어있거나, 시간 포맷이 규격에 맞지 않는 경우 **400 Bad Request** 반환.
    
- **가용성 불일치 (Availability Mismatch) 경고:** 사용자가 프론트엔드 모달에서 '시간 미세 조정'을 통해 임의로 `startTime` 및 `endTime`을 변경하여 전송할 수 있습니다. 이는 서버 알고리즘이 최초로 도출했던 가용성과 어긋나는(불일치하는) 임의 조작이므로, 최종 확정 데이터의 정합성 측면에서 반드시 "시간이 임의로 변경되었음"을 인지하고 조작해야 하는 비즈니스 룰을 내포하고 있습니다.
