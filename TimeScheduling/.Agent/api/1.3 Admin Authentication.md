### [API 1.3] Admin Authentication (POST /api/sessions/{sessionId}/verify-admin)

▼ **Header**

- `Content-Type: application/json` : mandatory
    

▼ **Request**

- **Path Variable**
    
    - `sessionId` (string, UUID 형식): 관리자 권한을 검증하고자 하는 일정 조율 방의 고유 ID
        
- **Request Body (Payload)**
    
    - `adminKey` (string, UUID 형식): 클라이언트(Client)가 모달창에서 입력한 관리자 인증 키
        

JSON

```
{
  "adminKey": "79ed427a-f75c-499d-9f4d-eb07cdfe9859"
}
```

▼ **Response**

- 검증 결과 데이터 (object)
    
    - `isValid` (boolean): Backend에서 DB의 실제 관리자 키와 비교(Validation)하여 일치하면 `true`, 불일치하면 `false` 반환
        

JSON

```
{
  "isValid": true
}
```

▼ **Error**

- `errorCode` (string): 시스템 에러 코드
    
- `message` (string): 에러 상세 메시지
    

JSON

```
{
  "errorCode": "ERR_SESSION_NOT_FOUND",
  "message": "존재하지 않거나 만료된 세션입니다."
}
```

▼ **예외 (Exception Handling)**

- **세션 부재 (Not Found):** 존재하지 않거나 유효기간 경과로 인해 데이터베이스에서 Purge(삭제)된 `sessionId`로 접근할 경우 404 Not Found 반환.
    
- **형식 오류 (Bad Request):** `adminKey`가 유효한 UUID 형식이 아니거나 Request Body가 누락된 경우 400 Bad Request 반환. (※ 인증 실패 자체는 에러가 아니며, `200 OK`와 함께 `"isValid": false`를 반환하는 것이 RESTful API 설계의 표준입니다.)
