
### [API 3.1] Meeting Schedule Recommendation (GET /api/sessions/{sessionId}/recommendations)

▼ **Description**

- 오직 `domainType`이 "MEETING"인 세션에서만 호출 가능한 Endpoint입니다.
    
- **Data Fetching:** UI에서 동적으로 필참 인원을 넘기지 않습니다. Backend의 `ScheduleRecommendationService`가 DB에 영속화(Persisted)된 `Participant` 엔티티의 `isMandatory` 값을 직접 Load하여 Bitwise operation(비트 논리 연산)에 반영합니다.
    

▼ **Header**

- (GET 요청이므로 별도의 Body Content-Type은 불필요하지만, 인증 토큰이 필요하다면 포함)
    

▼ **Request**

- **Path Variable**
    
    - `sessionId` (string, UUID 형식): 추천안을 조회하고자 하는 일정 조율 방의 고유 ID
        

▼ **Response**

- 전원의 가용 시간(Availability)을 Bitwise AND(`&=`) 연산하여 도출된 상위 교집합 추천 스케줄 배열 (array of objects)
    
    - `rank` (integer): 추천안 순위 (1~3위)
        
    - `type` (string): 추천 알고리즘 전략 타입 ("MAX_ATTENDANCE" 또는 "MAX_CONTINUITY")
        
    - `date` (string): 추천된 일정의 날짜 (Format: YYYY-MM-DD)
        
    - `startTime` (string): 추천 시작 시간 (Format: HH:mm)
        
    - `endTime` (string): 추천 종료 시간 (Format: HH:mm)
        
    - `attendanceRate` (string): 전체 인원 대비 참석 가능 비율 (예: "100%", 프론트엔드 UI 렌더링용)
        
    - `attendeesCount` (integer): 해당 시간에 참석 가능한 실제 멤버 수
        
    - `attendees` (array of objects): 해당 타임 슬롯에 참석 가능한 멤버 객체 명단 (동명이인 식별을 위한 고유 ID 포함)
        
        - `id` (string, UUID): 참가자 고유 식별자
            
        - `name` (string): 참가자 이름

▼ **Error**

- `errorCode` (string): 시스템 에러 코드
    
- `message` (string): 에러 상세 메시지
    

**JSON (예시: 잘못된 도메인 타입 접근 시)**

JSON

```
{
  "errorCode": "ERR_INVALID_DOMAIN_REQUEST",
  "message": "해당 세션은 근무(WORK) 도메인이므로 API 3.3 엔드포인트를 사용해야 합니다."
```

▼ **예외 (Exception Handling)**

- **세션 부재 (Not Found):** 존재하지 않는 `sessionId` 요청 시 **404 Not Found** 반환.
    
- **도메인 불일치 (Bad Request / Conflict):** DB 상의 `domainType`이 "WORK"인데 `[API 3.1]`을 호출한 경우 **400 Bad Request** 반환.
    
- **데이터 부족 (Unprocessable Entity):** 가용 시간을 제출한 멤버가 1명 이하이거나 교집합(Intersection)이 전혀 존재하지 않아 연산(Computation)이 불가능할 경우 빈 배열(`[]`)을 반환. (Client 측에서 "추천할 수 있는 스케줄이 없습니다" UI 처리)
